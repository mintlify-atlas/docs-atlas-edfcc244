## Overview

This guide will walk you through using Spec2Tools in all three modes: SDK, CLI, and MCP server. Choose the approach that fits your needs.

<Note>
  All examples assume you have an OpenAPI specification file (YAML or JSON) ready. You can use a local file path or a URL to a remote spec.
</Note>

## SDK Mode

Use Spec2Tools as a library with the Vercel AI SDK.

<Steps>
  <Step title="Install the SDK">
    Install the `@spec2tools/sdk` package along with the AI SDK and your preferred model provider:

    <CodeGroup>
      ```bash npm
      npm install @spec2tools/sdk ai @ai-sdk/openai
      ```

      ```bash yarn
      yarn add @spec2tools/sdk ai @ai-sdk/openai
      ```

      ```bash pnpm
      pnpm add @spec2tools/sdk ai @ai-sdk/openai
      ```
    </CodeGroup>
  </Step>

  <Step title="Set your API key">
    Set your OpenAI API key (or the key for whichever model provider you're using):

    ```bash
    export OPENAI_API_KEY=your-api-key-here
    ```
  </Step>

  <Step title="Create tools from your spec">
    Create a simple script that loads your OpenAPI spec and generates tools:

    ```typescript
    import { createTools } from '@spec2tools/sdk';
    import { generateText, stepCountIs } from 'ai';
    import { openai } from '@ai-sdk/openai';

    const tools = await createTools({ spec: './openapi.yaml' });

    const result = await generateText({
      model: openai('gpt-4o'),
      tools,
      prompt: 'List all users',
      stopWhen: stepCountIs(3)
    });

    console.log(result.text);
    ```

    <Tip>
      You can also pass a URL to `createTools()` instead of a file path:
      ```typescript
      const tools = await createTools({ 
        spec: 'https://api.example.com/openapi.json' 
      });
      ```
    </Tip>
  </Step>

  <Step title="Run your code">
    Execute your script:

    ```bash
    node your-script.js
    ```

    The AI will use the tools generated from your OpenAPI spec to fulfill the request.
  </Step>
</Steps>

### SDK with Code Mode

For large APIs, enable code mode to reduce token usage:

```typescript
import { createTools } from '@spec2tools/sdk';

const tools = await createTools({
  spec: './openapi.yaml',
  codeMode: true,
});
```

This collapses all endpoints into just 2 tools (`search` and `execute`), cutting token usage by ~99.9%.

<Note>
  The SDK only supports APIs **without authentication** when using `createTools()` directly. For authenticated APIs, use the CLI or MCP server, or enable code mode which handles authentication internally.
</Note>

## CLI Mode

Use the interactive CLI to chat with an AI agent that can call your API.

<Steps>
  <Step title="Set your OpenAI API key">
    The CLI uses OpenAI's GPT-4o model by default:

    ```bash
    export OPENAI_API_KEY=your-api-key-here
    ```
  </Step>

  <Step title="Start the agent">
    Run the CLI with your OpenAPI spec:

    ```bash
    npx @spec2tools/cli start --spec ./openapi.yaml
    ```

    You can also use a remote spec:

    ```bash
    npx @spec2tools/cli start --spec https://api.example.com/openapi.json
    ```
  </Step>

  <Step title="Chat with the agent">
    Once started, you can type natural language requests:

    ```
    > List all users
    > Create a user named John with email john@example.com
    > What endpoints are available?
    ```

    The agent will use the available tools to fulfill your requests.
  </Step>
</Steps>

### CLI with Authentication

If your API requires authentication, the CLI will prompt you interactively:

```bash
npx @spec2tools/cli start --spec ./openapi.yaml
```

Or provide the API key directly:

```bash
npx @spec2tools/cli start --spec ./openapi.yaml --api-key your-api-key
```

### CLI Special Commands

The CLI supports several special commands:

- `/tools` - List all available tools
- `/call <tool> [args]` - Call a tool directly
- `/schema <tool>` - Show tool parameter schema  
- `/clear` - Clear conversation history
- `/help` - Show help message
- `/exit` - Exit the CLI

**Example:**

```bash
> /call createUser --name "John" --email "john@example.com"
```

### CLI Code Mode

Enable code mode for large APIs:

```bash
npx @spec2tools/cli start --spec ./openapi.yaml --code-mode
```

## MCP Server Mode

Expose your API as an MCP server for Claude Desktop and other MCP clients.

<Steps>
  <Step title="Add to Claude Desktop">
    Use the `claude mcp add` command to configure the server:

    ```bash
    claude mcp add --transport stdio my-api \
      -- npx @spec2tools/stdio-mcp ./openapi.yaml
    ```

    Replace `my-api` with a name for your server, and `./openapi.yaml` with your spec path.
  </Step>

  <Step title="Restart Claude Desktop">
    After adding the server, restart Claude Desktop for the changes to take effect.
  </Step>

  <Step title="Use in Claude">
    Your API endpoints are now available as tools in Claude Desktop. Simply chat with Claude and it will use the tools as needed.
  </Step>
</Steps>

### MCP with Code Mode

For large APIs, enable code mode to dramatically reduce token usage:

```bash
claude mcp add --transport stdio my-api \
  -- npx @spec2tools/stdio-mcp ./openapi.yaml --code-mode
```

### MCP Manual Configuration

You can also manually configure the MCP server in Claude Desktop's config file (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "my-api": {
      "command": "npx",
      "args": ["@spec2tools/stdio-mcp", "./openapi.yaml"]
    }
  }
}
```

### MCP with Authentication

Provide authentication via environment variable or command-line flag:

```bash
claude mcp add --transport stdio my-api \
  -- npx @spec2tools/stdio-mcp ./openapi.yaml --api-key your-key
```

Or in the config file:

```json
{
  "mcpServers": {
    "my-api": {
      "command": "npx",
      "args": ["@spec2tools/stdio-mcp", "./openapi.yaml", "--api-key", "your-key"],
      "env": {
        "API_KEY": "your-key-here"
      }
    }
  }
}
```

## Converting Existing Tools to Code Mode

You can also convert any existing AI SDK tools to code mode:

```typescript
import { convertToolsToCodeMode } from '@spec2tools/sdk';
import { createMCPClient } from '@ai-sdk/mcp';

const client = await createMCPClient({
  transport: { type: 'sse', url: 'http://localhost:3000/sse' },
});

const tools = await client.tools();
const codeModeTools = convertToolsToCodeMode(tools);
```

This works with tools from any source, not just Spec2Tools.

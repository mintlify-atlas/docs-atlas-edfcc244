---
title: AuthManager
description: Manage authentication for API requests
---

The `AuthManager` class handles authentication flows including OAuth2, API keys, bearer tokens, and basic auth. It manages tokens and provides headers for authenticated requests.

## Constructor

Create a new AuthManager instance.

```typescript
new AuthManager(globalAuthConfig: AuthConfig)
```

<ParamField path="globalAuthConfig" type="AuthConfig" required>
  Global authentication configuration extracted from the OpenAPI spec
</ParamField>

**Example:**
```typescript
import { AuthManager } from '@spec2tools/core';

const authManager = new AuthManager({
  type: 'oauth2',
  authorizationUrl: 'https://api.example.com/oauth/authorize',
  tokenUrl: 'https://api.example.com/oauth/token',
  scopes: ['read', 'write']
});
```

## Methods

### requiresAuth

Check if authentication is required.

```typescript
requiresAuth(authConfig?: AuthConfig): boolean
```

<ParamField path="authConfig" type="AuthConfig">
  Optional tool-specific auth config that overrides the global config
</ParamField>

<ResponseField name="return" type="boolean">
  `true` if authentication is required, `false` if auth type is 'none'
</ResponseField>

**Example:**
```typescript
if (authManager.requiresAuth()) {
  await authManager.authenticate();
}
```

### getAuthHeaders

Get authorization headers for requests.

```typescript
getAuthHeaders(authConfig?: AuthConfig): Record<string, string>
```

<ParamField path="authConfig" type="AuthConfig">
  Optional tool-specific auth config that overrides the global config
</ParamField>

<ResponseField name="return" type="Record<string, string>">
  Object containing authorization headers. Empty object if no token is set.
</ResponseField>

**Header Formats:**
- **OAuth2/Bearer:** `{ Authorization: 'Bearer <token>' }`
- **Basic:** `{ Authorization: 'Basic <base64>' }`
- **API Key (header):** `{ [apiKeyHeader]: '<token>' }`

**Example:**
```typescript
const headers = authManager.getAuthHeaders();
// Returns: { Authorization: 'Bearer abc123...' }
```

### authenticate

Perform authentication based on the config type.

```typescript
async authenticate(authConfig?: AuthConfig): Promise<void>
```

<ParamField path="authConfig" type="AuthConfig">
  Optional tool-specific auth config that overrides the global config
</ParamField>

**Behavior by Auth Type:**

<Expandable title="OAuth2">
  1. Registers OAuth2 client dynamically at `/register` endpoint
  2. Starts local callback server on port 54321
  3. Opens browser for user authorization
  4. Exchanges authorization code for access token using PKCE
  5. Stores access token and optional refresh token
</Expandable>

<Expandable title="API Key">
  Prompts user to enter API key via stdin
</Expandable>

<Expandable title="Bearer Token">
  Prompts user to enter bearer token via stdin
</Expandable>

<Expandable title="Basic Auth">
  Prompts user for username and password, then base64 encodes credentials
</Expandable>

<Expandable title="None">
  No authentication performed
</Expandable>

**Throws:**
- `AuthenticationError` if authentication fails or times out
- `AuthenticationError` if required credentials are missing
- `AuthenticationError` if OAuth2 client registration fails
- `AuthenticationError` if token exchange fails

**Example:**
```typescript
try {
  await authManager.authenticate();
  console.log('Authentication successful');
} catch (error) {
  console.error('Authentication failed:', error.message);
}
```

## OAuth2 Flow Details

The AuthManager implements OAuth2 Authorization Code flow with PKCE:

1. **Client Registration:** Automatically registers a client at `{tokenUrl}/register` with:
   - Client name: "OpenAPI Agent CLI"
   - Grant types: authorization_code
   - Token endpoint auth method: none (public client)
   - Redirect URI: `http://127.0.0.1:54321/callback`

2. **Authorization:** Opens browser to authorization URL with:
   - PKCE code challenge (SHA-256)
   - Requested scopes
   - State parameter for CSRF protection

3. **Callback:** Local server receives authorization code on port 54321

4. **Token Exchange:** Exchanges code for access token with PKCE verifier

5. **Storage:** Stores access token and optional refresh token

**Port Configuration:**
- Callback port: 54321
- Callback path: `/callback`
- Authorization timeout: 5 minutes
